🎞 Animation Philosophy
(Trauma-Informed, Accessible, Anti-Coercive Motion)
Core Orientation
Animation in Whelm is regulatory, not decorative.
Motion exists to:
●
Support orientation
●
Reduce cognitive load
●
Communicate transitions safely
●
Help the nervous system track change
Animation must never:
●
Create urgency
●
Demand attention
●
Reward speed or completion
●
Perform “delight” at the user’s expense
Trauma-Informed Motion Principles
1. Predictability over surprise
○
Motions should be learnable and repeatable
○
Avoid sudden entrances, exits, or transformations
2. Slowness without drag
○
Default timing errs on the slower side
○
Users should never feel rushed to “keep up”
3. Soft beginnings and endings
○
No hard snaps or sharp eases
○
Motion should feel like arriving and leaving, not appearing/disappearing
4. Motion as containment
○
Animations should help hold attention gently
○
Nothing should feel like it’s pulling the user forward
Timing Guidelines (Conceptual, Not Prescriptive)
●
Avoid micro-animations designed to stimulate dopamine
●
Prefer longer, fewer transitions over many small ones
●
If a motion draws attention, it should also reduce choice or complexity
When in doubt:
Slow down, soften, or remove the animation.
Easing Philosophy
●
Use organic, non-mechanical easing curves
●
Avoid bouncy, elastic, or overshooting eases
●
Motion should resemble breath, gravity, or settling — not springiness
Easing should communicate:
●
Continuity
●
Stability
●
Grounding
Not:
●
Excitement
●
Reward
●
Gamification
Accessibility & Inclusion Commitments
●
All motion must respect prefers-reduced-motion
●
Reduced motion is not a degraded experience — it is a first-class path
●
No meaning should be conveyed only through motion
●
Users must not be required to visually track moving elements to proceed
Anti-Capitalist & Anti-Coercive Motion
Animation must not:
●
Imply progress as virtue
●
Frame speed as success
●
Create visual “pressure” to continue
●
Signal that stopping is interruption or failure
There should be no visual punishment for:
●
Pausing
●
Skipping
●
Leaving mid-flow
What Is Stable vs Emerging
Stable (Non-Negotiable):
●
Motion supports regulation
●
Motion never creates urgency
●
Motion respects accessibility preferences
●
Motion does not function as reward
Emerging Through Build:
●
Exact timing values
●
Specific easing curves
●
Where motion adds clarity vs where stillness is better
These will continue to be tested against real nervous-system response, not aesthetic trends.
Review Rule for New Animations
Before merging any new animation, ask:
Does this help the user feel more oriented — or more impressed?
If it’s the latter, it does not belong here.
